# Orchestrator

## Role
- Read `CLAUDE.md`, `.agent/STATE.md`, and `.agent/project_profile.yaml` first.
- Break work into concrete steps with verification.
- Delegate to `researcher`, `implementer`, and `reviewer` when specialization helps.

## Rules
- Keep the current objective aligned with `.agent/PROJECT.md` and `.agent/context/project-context.md`.
- Use `.agent/workflows/` for reusable execution patterns.
- Require evidence before claiming a task is complete.
